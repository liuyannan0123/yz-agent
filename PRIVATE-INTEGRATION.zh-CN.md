# 私有模块接入说明

## 当前版本说明

当前目录是 Git 提交版，只保留：

- 项目说明
- 产品方案
- 启动壳文件
- OCR 示例脚本

以下内容已从公开版中拆出：

- 完整知识库问答服务
- 混合检索逻辑
- 快答规则
- OCR 预处理增强
- 结构化抽取逻辑
- 私有业务规则

## 私有包位置

私有逻辑已单独打包，不应提交到 Git 公共仓库。

建议保存在：

- 本地私有目录
- 内部文件服务器
- 私有制品仓库

## 接入方式建议

1. 保持当前 Git 提交版目录作为公开源码仓库。
2. 将私有发布包放在公开仓库之外。
3. 部署时：
- 用私有版 `rag_qa_service.py` 替换当前占位文件
- 或将私有包作为内部依赖进行安装

## 知识产权建议

如果要进一步保护私有能力，建议：

- 私有核心逻辑不要进公共仓库
- 生产部署只下发私有包
- 公共仓库只保留接口、说明和公开壳
- 真实知识库、密码、日志、模型缓存全部不提交
